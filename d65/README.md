# 2D 平台跳跃游戏 - 外部关卡编辑器

这是一个使用 Godot 4 开发的 2D 平台跳跃游戏，配合 Python tkinter 编写的外部关卡编辑器，通过 TCP 通信实现实时关卡加载和双向消息传递。

## ✨ 最新功能：双向通信

### 金币收集同步
- **Godot → Python**：玩家吃到金币时，游戏发送消息到编辑器
- **视觉反馈**：Python 编辑器中对应的金币自动变灰（#888888）
- **消息格式**：`{"event": "coin_collected", "id": "coin_1"}`

## ✅ 修复内容

### Godot 端修复
1. **使用正确的 Godot 4 API**：`TCP_Server` 作为服务器
2. **非阻塞数据读取**：在 `_process` 中使用 `poll()` + `get_partial_data()`
3. **正确的 JSON 解析**：使用 `JSON.new()` 对象 + `parse()` 方法
4. **添加错误处理**：详细的 JSON 解析错误信息
5. **类型安全**：使用 `Dictionary` 类型注解 + `is` 类型检查
6. **双向通信**：添加 `send_message()` 方法发送消息到 Python

### Python 端修复
1. **长连接保持**：建立连接后保持，不重复新建
2. **自动重连**：连接断开后自动尝试重新连接
3. **后台监听**：独立线程持续监听来自游戏的消息
4. **实时反馈**：金币被收集后编辑器中立即变灰
5. **状态显示**：界面显示连接状态（已连接/等待连接/连接断开）

## 功能说明

### Godot 游戏
- **玩家角色**：CharacterBody2D，支持左右移动和跳跃
- **控制键**：
  - A/←：向左移动
  - D/→：向右移动
  - W/空格/↑：跳跃
- **TCP 服务器**：监听 localhost:8888，双向通信
- **动态生成**：接收到数据后动态生成平台、敌人、金币
- **金币收集**：玩家碰到金币时发送消息通知编辑器

### Python 关卡编辑器
- **GUI 界面**：使用 tkinter 构建
- **元素类型**：
  - 平台（绿色）：StaticBody2D，可站立
  - 敌人（红色）：CharacterBody2D
  - 金币（黄色）：Area2D，收集后编辑器中变灰
- **操作功能**：
  - 点击画布创建元素
  - 拖拽移动元素位置
  - 删除选中元素
  - 清空画布
  - 发送关卡数据到游戏
- **实时同步**：接收金币收集消息并更新显示

## 使用方法

### 重要：启动顺序
**必须先启动 Godot 游戏**，再启动 Python 编辑器！

### 1. 启动 Godot 游戏
1. 打开 Godot 4 编辑器
2. 导入 `godot_game` 目录作为项目
3. 点击运行按钮（或按 F5）
4. 控制台应显示："TCP 服务器已启动，监听端口 8888"

### 2. 启动 Python 关卡编辑器
```bash
python level_editor.py
```
- 编辑器会自动尝试连接游戏
- 状态栏显示连接状态

### 3. 编辑和发送关卡
1. 在 Python 编辑器中选择元素类型
2. 点击画布放置元素（建议放置金币测试同步功能）
3. 拖拽调整元素位置
4. 点击"发送到游戏"按钮
5. 在 Godot 游戏中查看生成的关卡

### 4. 测试双向通信
1. 控制玩家角色（蓝色方块）走向金币
2. 接触金币后，游戏中的金币消失
3. **同时**：Python 编辑器中对应的金币会变灰！
4. 查看两端控制台输出确认消息传递

## 通信协议

### Python → Godot（关卡数据）
```json
[
  {"type": "platform", "x": 200, "y": 500, "width": 100, "height": 50},
  {"type": "enemy", "x": 400, "y": 500, "width": 50, "height": 50},
  {"type": "coin", "x": 300, "y": 400, "width": 40, "height": 40}
]
```

### Godot → Python（事件消息）
```json
{"event": "coin_collected", "id": "coin_1"}
```

## 常见问题

**Q: 点击"发送到游戏"提示连接失败？**
A: 确保 Godot 游戏正在运行，并且先启动 Godot 再启动 Python。

**Q: 金币被收集后编辑器没有变灰？**
A: 检查两端控制台输出，确认：
1. Godot 端显示"发送消息: {...}"
2. Python 端显示"收到消息: {...}"

**Q: 连接状态显示"连接断开"？**
A: 编辑器会自动尝试重连，或重启 Python 编辑器重新连接。
